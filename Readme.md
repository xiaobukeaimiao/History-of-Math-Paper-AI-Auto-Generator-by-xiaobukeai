# AI Paper Generator 使用文档

---
## 0. 介绍
这是一个利用 LLM API 自动生成超长 LaTeX 文本的自动化脚本。

如果您被要求完成 **50 页**的论文的话，这个工具可能会比较有帮助。

## 1. 配置文件 (`config.json`)
在运行程序前，请先配置 `config.json` 中的以下字段（复制 `config.example.json` 文件并修改）：

* **api_key**: 填写你的 API 密钥。
* **base_url**: 对应的 API 接口请求地址。
* **model**: 要调用的模型名称。
* **debug_mode**: 调试模式，填写 `false` 即可。
* **global_prompt**: AI 的预设提示词，该内容会在每次对话中发送。

---

## 2. 论文大纲 (`outline.txt`)
论文大纲需遵循严格的语法格式，以便程序解析。

### 【输出格式要求】
1. **层级结构语法**：
    - 一级标题：使用 `section` + 标题内容。
    - 二级标题：使用 `subsection` + 标题内容。
    - 三级标题：使用 `subsubsection` + 标题内容。
2. **内容描述**：每个标题的下一行必须以 **`[描述]`** 开头，写明该章节的写作指导、核心论点及计划页数。

### 【AI 提示词例子】
以下提示词可以让 AI 为你生成符合格式的大纲（仅作为示例）：

>你是一位资深的数学史专家。我现在需要撰写一篇50页的数学史论文，题目是……。
>
>请为我生成一份详尽的、逻辑严密的论文大纲。
>
>【输出格式要求】（极为严格，请严格遵循）：
>
>1. 必须且只能使用以下的语法来表示层级结构。
>
>   - 一级标题使用 section，加上标题内容
>   - 二级标题使用 subsection，加上标题内容
>   - 三级标题使用 subsubsection，加上标题内容
>
>2. 在每一个标题的下一行，必须以“[描述]”开头，写出一段针对该章节/小节的详细写作指导和核心论点提示。
>
>3. 描述中要包含计划的页数，合起来要和计划的总页数相近
>
>4. 如果某一级标题下有子标题，该父级标题依然需要一段简短的“[描述]”，用于引出下文。
---

## 3. 主程序 (`AI Paper Generator.exe`)
* **运行**：配置好上述两个文件后，直接双击运行。
* **特性**：
    - 支持**断点续传**：如果遇到网络错误或手动打断，下次运行可继续生成。
    - 实时保存进度。

---

## 4. 输出结果 (`OUTPUT/`)
* 程序运行完成后，生成的 LaTeX 源代码文件 `Paper.tex` 将保存在 `OUTPUT` 文件夹下。

## 5. 提示
* 本项目生成的文本仅供技术测试，严禁直接用于实际的学术论文提交，作者不对任何学术不端行为负责。
* 输出的tex文件有可能过不了编译，一般来说报错不会很多，自行修改即可。
* 你可以在大纲最后增加一节"参考文献"。
* 以 2026 年 5 月的 Deepseek V4 Pro 的 API 价格，生成 50 页论文的成本约为 1 元。
